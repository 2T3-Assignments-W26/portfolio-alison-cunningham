## The Paddle Game:
This is a console-based paddle game built in C#.

The player uses the Up and Down arrow keys to control a verticle paddle on the right of the gameboard. They are playing against an **AI-controlled opponent** the left. A ball bounces around the play area, and the goal is to hit the ball past your opponent’s paddle to score points. The first player to reach **3 points** wins the match.

>Players can choose from 3 difficulty levels: Easy, medium, or hard.

![welcome screen for the paddle game](../photos_assignment1/paddle_welcome.png)

Each time a point is score by either player, it unlocks a **random game mode** for the next round. Random game modes include:
- **Ghost Ball** – The ball briefly disappears at intervals.
- **Fast Ball** – The ball moves faster than normal.
- **Small Paddle** – The paddles become shorter.
- **Drunk Paddle** – The player’s paddle may randomly move on its own.
- **Curve Ball** – The ball’s vertical direction randomly shifts as it moves.

![paddle game gameplay](../photos_assignment1/paddle_gameplay.png)

Difficulty levels were implemented by programming the AI-controlled opponent to have variable accuracy based on the selected setting. The lower the difficulty, the more often the opponent will miss the ball.

```C#
if (CurrentDifficulty == 1) HitOrMiss = r.Next(0, 2);
if (CurrentDifficulty == 2) HitOrMiss = r.Next(0, 5);
if (CurrentDifficulty == 3) HitOrMiss = r.Next(0, 9);
```
If the random number generated each iteration through the game loop is a 0, the opponent will miss the ball.

This resulted in the following miss rate:

>Difficulty | Miss frequency
>|----|----|
>Easy | 50% 
>Medium | 20% 
>Hard |~11%

# Five Mini-Programs

This is a menu-driven C# console application that allows the user to select from **five mini-programs**:

1. **Area of Rectangles** – Calculate and compares the areas of two rectangles based on user-provided lengths and widths.
   
2. **Collatz Conjecture** – Applies the Collatz sequence to a valid number input, and tracks iterations and how often numbers are above or below 100.

3. **Dollar Game** – Users enter quantities of pennies, nickels, dimes, and quarters until the total exactly $1.

4. **Oldest Person** – The user inputs the names and ages of multiple people and the program identifies the oldest one.

5. **Hi Lo Game** – A guessing game where the program chooses a random number between 1 and 100, and the user guesses until correct.

The program uses **input validation**, **loops**, and **conditional logic** to interact with the user.

The program uses **TryParse** to test for valid input. If **TryParse** returns *false*, the programs outputs an error message, prompting the user to input another valid number.

```C#
do
{
    goodinput = Int32.TryParse(Console.ReadLine(), out goodnum);

    if (!goodinput)
    {
        Console.WriteLine("You gave an invalid number.");
        Console.WriteLine();
        Console.WriteLine("Please try again");
        Console.WriteLine();
    }

} while (!goodinput);

return goodnum;
```

### What I learned
Through this project, I learned how to build a robust menu system using a **switch statement**, how to use **loops** and **conditionals** effectively, and how to use **input validation** to prevent errors. Overall, I gained experience at building an interactive and user-friendly program in C#.
